# 프라이버시 vs 편의성 & 동기화 계획

> 일시: 2026-02-05

## 논의 배경

AI 기반 자동 정리(Zero Guilt)를 구현하려면 서버로 데이터를 보내야 함.
→ 프라이버시 우려 가능성에 대한 논의.

---

## 1. 문제 제기

"정리할 때 AI를 활용하려면 서버를 거쳐야 하는데, 유저의 메모가 서버에 저장된다면 사람들이 우려할 수 있지 않을까?"

---

## 2. 검토한 옵션들

| 옵션 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **On-device AI** | 기기에서 직접 처리 (ML Kit, Gemma Nano 등) | 프라이버시 완벽 보장 | 성능 제한, 복잡한 분류 어려움 |
| **서버 처리 + 투명한 동의** | 서버로 전송하되 명확히 고지 | 강력한 AI 활용 가능 | 일부 사용자 이탈 |
| **하이브리드** | 간단한 건 로컬, 고급 기능만 서버 | 균형 | 복잡도 증가 |

---

## 3. 결정

### 타겟 유저

**편의성 우선인 사람들**

- "우려할 수 있다"와 "실제로 문제다"는 다르다
- Notion, Evernote도 다 서버에 저장하는데 잘 쓴다
- 프라이버시에 극도로 민감한 사람은 타겟이 아님

### 최종 방향

| 항목 | 방향 |
|------|------|
| **디폴트** | AI 분류 ON (편의성 우선) |
| **설정** | AI 분류 OFF 옵션 제공 (프라이버시 우선 사용자용) |
| **서버 저장** | 숨기지 말고 명확히 고지 |

---

## 4. 동기화 & 확장 계획

### 계획 중인 기능

- PC 동기화
- Obsidian 연동
- 멀티디바이스 지원

### 서버 저장 필수성

> "동기화 기능 추가해서 PC로 확장할 생각도 하는데, 어차피 그거 하려면 서버 저장을 안할 수가 없다"

동기화/멀티디바이스를 고려하면 서버 저장은 선택이 아닌 필수.
프라이버시 우려보다 기능적 이점이 더 크다고 판단.
