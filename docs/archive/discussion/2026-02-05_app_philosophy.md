## 1. 핵심 철학: 최소한의 Friction

### Friction의 정의

**"의도와 행동 사이의 저항"**

- 인지적 부담이 핵심 (뭘 선택해야 하지?)
- 물리적 단계(탭 수)도 결국 인지적 부담을 유발
- 선택지가 많을수록 friction 증가

### 어떤 순간의 Friction을 줄여야 하는가?

**결론: 캡처 순간이 가장 중요하다**

이유:
1. 아이디어는 휘발성이 있다 - 정리는 나중에 해도 되지만, 캡처를 놓치면 아이디어 자체가 사라짐
2. Flit.의 진짜 경쟁자는 다른 메모앱이 아니라 "나중에 적어야지" 하다가 잊어버리는 것
3. friction이 조금이라도 있으면 사람들은 그냥 넘어간다

## 2. Guilt-Free Capture

### 세 가지 Zero

| 원칙 | 설명 |
|------|------|
| **캡처의 Zero Friction** | 앱 실행 → 바로 입력 가능. 이 순간이 가장 frictionless해야 함 |
| **정리의 Zero Guilt** | 사용자는 던지기만 하면 됨. 분류/태깅은 AI가 알아서 처리. "나중에 정리해야지"라는 부채감 자체를 없앤다 |
| **실행의 Zero Effort** | 일정 → 캘린더, 할 일 → 투두 등 실제 action까지 AI가 대신 처리. 기록에서 멈추지 않고 행동까지 연결 |

### 전체 흐름

```
아이디어 떠오름 → 앱 열고 바로 입력 → AI가 분류 → AI가 action까지 실행
                 (Zero Friction)      (Zero Guilt)   (Zero Effort)
```

**"던지기만 하면 진짜 끝"** - 기록만 하는 게 아니라 실제 필요한 곳에 반영까지

### Flit.에서 가장 frictionless해야 하는 순간

> 앱을 처음 실행했을 때 바로 메모 가능하도록 하는 부분
