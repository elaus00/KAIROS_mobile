## P0 — 치명적 이슈

### P0-1. 분석 대시보드 (AnalyticsDashboard) 완전 공백
- **스크린샷**: `analytics_dashboard_default.png`
- **현상**: "← 분석" 타이틀만 표시되고 본문 영역 전체가 빈 화면
- **문제**: 로딩 인디케이터도, 빈 상태 메시지도, 콘텐츠도 없음
- **기대**: 최소한 빈 상태 ("아직 분석할 데이터가 없습니다") 또는 로딩 UI 표시
- **대상 파일**: `AnalyticsDashboardScreen.kt`, `AnalyticsDashboardScreenshotTest.kt`

### P0-2. 캡처 상세 로딩 상태 — 인디케이터 미표시
- **스크린샷**: `capture_detail_loading.png`
- **현상**: "← 캡처 상세" 타이틀 + 화면 중앙에 작은 점 하나만 표시
- **문제**: CircularProgressIndicator가 아닌 아주 작은 점(dot)만 보임. 사용자가 로딩 중임을 인지하기 어려움
- **기대**: 명확한 로딩 인디케이터 (원형 스피너) + "로딩 중..." 텍스트
- **대상 파일**: `CaptureDetailScreen.kt`

- **수정안**: "Inbox" → "받은함" (앱 전체에서 사용하는 한국어 용어)

### P1-3. 로그인 화면 — 브랜드 태그라인 불일치

- **불일치**: PRD/온보딩의 핵심 태그라인 "적으면, 알아서 정리됩니다"와 다름
- **수정안**: "적으면, 알아서 정리됩니다" 또는 "떠오른 순간, 바로 던지면 끝"
- **대상 파일**: `LoginScreen.kt`

## P2 — 패턴 일관성

### P2-1. 빈 상태(Empty State) 패턴 불일치
현재 화면별 빈 상태 구현 방식이 제각각:

| 화면 | 아이콘 | 제목 | 부제 | 평가 |
|------|--------|------|------|------|
| 노트 | ✅ | ✅ "아직 노트가 없습니다" | ✅ 안내 문구 | 모범 |
| 검색 | ✅ | ✅ "캡처를 검색하세요" | ❌ | 양호 |
| 휴지통 | ❌ | ✅ "휴지통이 비어 있습니다" | ❌ | 미흡 |
| 캘린더 | ❌ | ✅ "일정이 없습니다" | ❌ | 미흡 |
| 분석 | ❌ | ❌ | ❌ | **결여** |

**수정 방향**: 공통 EmptyStateView 컴포넌트 패턴 통일
- 구성: 아이콘 + 주 메시지 + (선택) 부 메시지
- 최소: 아이콘 + 주 메시지

### P2-2. 로딩 인디케이터 패턴 불일치

| 화면 | 표현 | 평가 |
|------|------|------|
| 캡처 상세 (loading) | 아주 작은 점(dot) | **미흡** |
| 히스토리 | 작은 점 + "로드 중..." | 미흡 |
| 재구성 (loading) | 점 + "AI가 노트를 분석 중입니다..." | 양호 |
| 캡처 (submitting) | 텍스트 변경 + 버튼 내 dot | 양호 |

**수정 방향**: 전체 화면 로딩 시 CircularProgressIndicator + 설명 텍스트 통일

### P2-3. 히스토리 화면 — 칩 행(row) 구분 혼란
- **스크린샷**: `history_default.png`
- **현상**: 유형 필터(전체/일정/할 일/노트)와 기간 필터(전체/오늘/...)가 동일한 칩 스타일로 한 줄에 나열
- **문제**: 두 필터 그룹의 시각적 구분 없이 같은 스타일 칩이 연속됨. "전체" 칩이 두 번 등장하여 혼란
- **수정안**: (A) 기간 필터에 다른 스타일 적용, 또는 (B) 구분선/간격 추가, 또는 (C) 섹션 레이블 추가
- **대상 파일**: `HistoryScreen.kt`

### P2-4. 캡처 제출 상태 — 피드백 부족
- **스크린샷**: `capture_submitting.png`
- **현상**: "제출 중인 캡처" 텍스트만 표시. 전송 버튼에 작은 점이 있으나 명확한 로딩 애니메이션 없음
- **문제**: 사용자가 제출 진행 상황을 직관적으로 인지하기 어려움
- **수정안**: 텍스트 입력 영역에 진행 표시 또는 전송 버튼을 CircularProgressIndicator로 교체
- **대상 파일**: `CaptureContent.kt`

### P2-5. 캡처 상세 vs 노트 상세 — 앱바 액션 비대칭
- **캡처 상세**: ← 제목 / 공유(⎋) — 2개 액션
- **노트 상세**: ← / 공유(⎋) 폴더(📁) 삭제(🗑) — 4개 액션
- **관찰**: 캡처 상세에는 삭제 버튼 없음. 사용자가 잘못 캡처한 것을 상세 화면에서 바로 삭제할 수 없음
- **검토 필요**: 의도된 설계인지, 캡처 상세에도 삭제 액션 필요한지 확인

## P3 — 시각적 개선

### P3-1. 구독 관리 (Free) — 기능 목록 시각 밀도 부족
- **스크린샷**: `subscription_free_tier.png`
- **현상**: Premium 기능 8개가 텍스트만으로 나열. 아이콘/체크마크 없음
- **비교**: Premium 티어는 체크마크(✓)가 있어 활성 상태가 명확
- **수정안**: Free 티어에서도 잠금(🔒) 아이콘 또는 비활성 표시로 "아직 안 됨" 시각화
- **대상 파일**: `SubscriptionScreen.kt`

### P3-2. 캘린더 섹션 — 빈 상태 시각적 밀도
- **스크린샷**: `calendar_default.png`
- **현상**: "일정이 없습니다" / "할 일이 없습니다" 텍스트만 표시
- **수정안**: 가벼운 일러스트레이션 또는 아이콘 추가로 빈 상태를 더 명확히 전달
- **대상 파일**: `CalendarScreen.kt`

### P3-3. 설정 화면 — 섹션 간 그룹핑 개선
- **스크린샷**: `settings_default_not_logged_in.png`
- **관찰**: 화면/글씨 크기/노트 보기가 하나의 카드에 구분 없이 들어감. 테마·글씨·보기 3개는 성격이 다른 설정
- **수정안**: 섹션 내 서브 헤더 또는 미세한 구분선으로 논리적 그룹 구분
- **대상 파일**: `SettingsScreen.kt`

### P3-4. 재구성 제안 카드 — 정보 밀도 향상
- **스크린샷**: `reorganize_with_proposals.png`
- **현상**: "3개 노트 / CREATE → 업무" 형태로 최소 정보만 표시
- **수정안**: 포함되는 노트 제목 미리보기(1~2개) 표시로 사용자가 제안 내용을 판단할 수 있도록 개선
- **대상 파일**: `ReorganizeScreen.kt`

## 코드 레벨 검증 필요

스크린샷만으로 확인 불가하여 코드 리뷰가 필요한 항목:

### V1. Color 토큰 사용 여부
- **기준**: `Color(0xFF...)` 하드코딩 금지, `FlitTheme.colors.*` 토큰만 사용
- **검증 방법**: `Color(0x` 패턴 Grep 검색
- **대상**: 전체 presentation/ 디렉토리

### V2. 터치 타겟 48dp 최소 준수
- **기준**: 모든 인터랙티브 요소의 터치 타겟 ≥ 48dp
- **검증 방법**: `.clickable` / `IconButton` / `Button` 컴포넌트의 size/minimumInteractiveComponentSize 확인
- **우려 화면**: 캡처 화면 상단 아이콘 바 (알림/히스토리/설정 아이콘이 밀집)

### V3. 8dp 그리드 정합성
- **기준**: padding/margin 값이 8의 배수 (예외: 4dp 미세 조정 허용)
- **검증 방법**: `.padding()` 호출의 dp 값 검증
- **대상**: 전체 화면
