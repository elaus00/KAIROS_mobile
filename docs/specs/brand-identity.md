## 1. 브랜드 에센스

### 1.1 브랜드 이름 해석

**Flit** [flɪt]
- (동사) 재빠르게 날아다니다, 가볍게 스치듯 지나가다
- (명사) 빠른 움직임, 순간적인 이동

"Flit."에서 마침표(.)는 **완결**을 의미한다.
- 떠오른 생각을 바로 던지면 → **끝** (Just Capture)
- AI가 분류하면 → **끝** (Auto Organize)
- 행동으로 연결되면 → **끝** (Straight to Action)

**브랜드 본질**: 생각의 빠른 포착과 자동 완결

### 1.2 핵심 키워드

| 키워드 | 설명 |
|--------|------|
| **즉각성 (Immediacy)** | 떠오른 순간 바로 기록. 판단·분류·정리를 미루지 않는다 |
| **경량성 (Lightness)** | 무거운 UX, 복잡한 계층, 정리 부담이 없다. 사용자는 가볍게 던지기만 하면 된다 |
| **정확성 (Precision)** | AI 분류는 정확해야 한다. 신뢰할 수 없는 자동화는 무의미하다 |
| **완결성 (Completeness)** | 기록 → 정리 → 실행이 자동으로 연결된다. 사용자는 중간에 방치하지 않는다 |
| **명료성 (Clarity)** | 모든 인터페이스는 즉시 이해 가능해야 한다. 모호함은 마찰이다 |

### 1.3 브랜드 퍼소나

**Flit.은 신뢰할 수 있는 자동화 비서다.**

- **효율적이되 차갑지 않다**: 기계적 자동화가 아니라, 사용자를 이해하고 돕는다
- **조용하되 무능하지 않다**: 화려한 UI로 주목받지 않지만, 핵심 작업(분류)에서 뛰어나다
- **단순하되 제한적이지 않다**: 인터페이스는 미니멀하지만, 복잡한 상황(다중 캘린더, 충돌 해결)도 처리한다
- **빠르되 조급하지 않다**: 캡처는 즉각적이지만, 사용자를 재촉하지 않는다

## 2. 로고 확정안

### 2.1 로고 시스템 개요

Flit. 로고는 **타이포그래피 워드마크 + 채움 도트**로 구성된다.
마침표(.)를 독립적인 시각 요소(filled circle)로 분리하여 "완결"의 의미를 강조한다.

### 2.2 워드마크 스펙

| 속성 | 값 | 비고 |
|------|------|------|
| **폰트** | Sora (Google Fonts) | 기하학적 산세리프 |
| **Weight** | 480 | Regular(400)~Medium(500) 사이, 가변 폰트 |
| **기본 크기** | 44px | 컨텍스트별 비례 조정 |
| **자간** | 1px | letter-spacing |
| **케이스** | Flit | 첫 글자만 대문자 |

### 2.3 채움 도트 스펙

| 속성 | 값 | 비고 |
|------|------|------|
| **형태** | filled circle | border-radius: 50% |
| **크기** | 7.5px | 44px 기준, 텍스트 높이 ~17% |
| **간격** | 2.5px | margin-left, 텍스트 바로 뒤 |
| **수직 위치** | 13px | position:relative; top:13px (베이스라인 정렬) |

### 2.4 스케일링 규칙

도트와 간격은 폰트 크기에 비례하여 조정한다:

| 폰트 크기 | 용도 | 도트 | 간격 | 위치(top) |
|-----------|------|------|------|-----------|
| 64px | 히어로 | 11px | 3.5px | 19px |
| 44px | 기본 (워드마크) | 7.5px | 2.5px | 13px |
| 32px | 스플래시 | 5.5px | 2px | 10px |
| 28px | 헤더 | 5px | 1.5px | 8px |
| 20px | 네비게이션/앱바 | 3.5px | 1px | 6px |
| 14px | 최소 | 2.5px | 0.5px | 4px |

### 2.5 앱 아이콘 (IC-5a)

검정 배경 위 흰색 F + 채움 도트. 워드마크와 별도의 고정 스펙을 갖는다.

| 속성 | 값 |
|------|------|
| **배경** | #111111 |
| **F 폰트** | Sora, weight 450 |
| **F 크기** | 28px (64px 아이콘 기준) |
| **F 색상** | #FFFFFF |
| **도트** | 6px filled circle |
| **도트 색상** | #FFFFFF |
| **도트 간격** | 1px |
| **도트 위치** | top 8px |

아이콘 사이즈별 비례:

| 아이콘 크기 | F 크기 | 도트 | border-radius |
|------------|--------|------|---------------|
| 96px | 40px | 8px | 22px |
| 80px | 34px | 7px | 18px |
| 64px | 28px | 6px | 14px |
| 48px | 20px | 4.5px | 11px |
| 36px | 16px (weight 500) | 3.5px | 8px |

### 2.6 설계 근거

- **채움 도트 선택 이유**: "Flit."은 마침표가 포함된 브랜드명. 채움 도트가 period로 읽혀 "적으면 끝"이라는 완결의 메시지를 시각적으로 완성함
- **IC-5a(반전) 선택 이유**: 모노크롬 검정 배경이 컬러풀한 경쟁 앱 사이에서 역으로 차별화됨. 작은 사이즈에서도 높은 대비로 인식성 보장
- **Sora 폰트 선택 이유**: 기하학적이고 모던한 산세리프. 가변 폰트로 480 같은 미세 weight 조정 가능
- **독립 점 사례 희소성**: 리서치 결과, 단일 채움 도트를 독립 시각 요소로 사용한 유명 브랜드는 극히 드묾 → 차별화 포인트

## 3. 컬러 전략

### 3.1 기존 시스템과의 관계

Flit. 디자인 시스템은 **미니멀리스트 모노크롬** 기조를 채택했다.
- 무채색 (회색 스케일)이 기본
- 유채색은 오직 **danger(빨강)/success(녹색)/warning(황색)** 3가지만 허용

로고는 이 모노크롬 철학을 존중해야 한다.

### 3.2 로고 자체 컬러 방향

**방향 1: 완전 모노크롬 유지 (권장)**

로고는 라이트/다크 모드에 따라 색상을 반전한다:
- **라이트 모드**: 검은색(#111111) 로고 on 흰색 배경
- **다크 모드**: 흰색(#FFFFFF 80% opacity) 로고 on 검은색(#0A0A0A) 배경

**장점**:
- 디자인 시스템과 완벽히 일관됨
- 스플래시, 앱 아이콘, 로딩 스피너 등 모든 맥락에서 재사용 가능
- 타임리스(timeless) — 트렌드에 영향받지 않음

**단점**:
- 브랜드 차별성이 약할 수 있음 (무채색 로고는 흔함)

---

**방향 2: 액센트 컬러 도입 (조건부)**

로고의 특정 요소(예: Motion Trail의 경로선 하나, 또는 Dot in Circle의 중심 점)에만 액센트 컬러를 추가한다.
- 액센트 컬러: **#FFFFFF 80% opacity** (다크 모드) / **#111111** (라이트 모드) — 즉, 모노크롬 강조색
- 또는 **#F59E0B (warning 황색)**을 매우 제한적으로 사용 (예: 로고 중심 점 1개만)

**장점**:
- 시각적 포인트 생성, 브랜드 기억성 증가
- 모노크롬 기조를 유지하면서 차별화 가능

**단점**:
- 황색은 danger/success/warning 중 "경고" 의미 — 긍정적 브랜드 뉘앙스와 맞지 않을 수 있음
- 일관성 관리 복잡도 증가

---

### 3.3 권장 전략

**방향 1 (완전 모노크롬 유지)**을 권장한다.

**이유**:
1. "명료성(Clarity)" 철학과 일치 — 색상은 의미 전달 수단이며, 장식이 아니다
2. 디자인 시스템 일관성 극대화
3. Adaptive Icon, 스플래시, 로딩 등 모든 맥락에서 통일된 브랜드 경험 제공
4. Apple/Google의 미니멀 디자인 트렌드와 정렬

---

### 3.4 Light/Dark 모드 대응 전략

| 요소 | 라이트 모드 | 다크 모드 |
|------|------------|----------|
| 워드마크 + 도트 | #111111 | rgba(255,255,255, 0.85) |
| 워드마크 배경 | #FAFAFA 또는 투명 | #0A0A0A 또는 투명 |
| 앱 아이콘 배경 | #111111 (반전) | #111111 (동일) |
| 앱 아이콘 F + 도트 | #FFFFFF | #FFFFFF |
| 스플래시 배경 | #FAFAFA | #0A0A0A |

Adaptive Icon 레이어:
- **Foreground**: F + 채움 도트 (흰색, 투명 배경)
- **Background**: 단색 #111111
- **Monochrome**: F + 채움 도트 (시스템 색상 적용)

## 4. 타이포그래피 (확정)

### 4.1 워드마크 서체

| 속성 | 확정 값 |
|------|---------|
| **서체** | Sora (Google Fonts, OFL 라이선스) |
| **Weight** | 480 (가변 폰트, Regular~Medium 사이) |
| **자간** | 1px (letter-spacing) |
| **케이스** | Flit (첫 글자 대문자) |

**Sora 선택 근거**:
- 기하학적 산세리프: "명료성" 철학과 일치
- 가변 폰트: 480처럼 미세한 weight 조정 가능
- Google Fonts 무료 라이선스 (OFL)
- 깔끔한 기하학적 형태, 4글자에서 균형감 우수

### 4.2 마침표(.) 처리 — 독립 채움 도트

마침표를 폰트 내장 글리프가 아닌 **독립적인 채움 원형(filled circle)**으로 분리한다.

```
Flit + ● (별도의 시각 요소)
```

- 크기: 텍스트 높이의 ~17% (44px 기준 7.5px)
- 위치: 베이스라인 정렬 (텍스트 우측, top:13px)
- 간격: 2.5px (margin-left)
- 형태: 완전한 원 (border-radius: 50%, filled)

**독립 도트 근거**:
- "완결" 개념을 시각적으로 강조
- 아이콘에서 F + 도트로 자연스럽게 분리 가능
- 단일 채움 도트를 독립 요소로 사용한 유명 사례가 희소 → 차별화

## 5. 태그라인 후보

### 5.1 한국어 태그라인

| 태그라인 | 기반 원칙 | 설명 |
|---------|----------|------|
| **떠오른 순간, 바로 던지면 끝** | Just Capture, Friction | 캡처의 즉각성과 단순함 강조 |
| **생각은 가볍게, 정리는 자동으로** | Lightness, Auto Organize | 사용자 부담 제거 + AI 자동화 |
| **기록 그 이상, 완결까지** | Completeness, Straight to Action | 기록 → 행동 자동 연결 |

---

### 5.2 영어 태그라인

| 태그라인 | 기반 원칙 | 설명 |
|---------|----------|------|
| **Capture. Organized. Done.** | Just Capture, Auto Organize, Completeness | 3단계 자동 흐름 간결하게 표현 |
| **Thoughts fly, AI sorts.** | Lightness, Auto Organize | "Flit" = 날아다니다 + AI 분류 |
| **From thought to action, instantly.** | Straight to Action, Immediacy | 생각 → 행동 즉각 연결 |

---

### 5.3 확정 태그라인

**한국어**: "적으면, 알아서 정리됩니다"
- 사용자 행동(적다) → AI 자동화(알아서 정리) 흐름을 자연스럽게 표현
- 일상 언어로 진입 장벽 낮춤

**영어**: "Just Capture. It organized."
- 의도적 문법 파격 ("It organized" = 마케팅적 허용)
- "Just"의 간결함 + 과거형의 완결감

## 6. 브랜드 보이스

### 6.1 톤 키워드

| 키워드 | 설명 |
|--------|------|
| **간결함 (Concise)** | 불필요한 말을 하지 않는다. 사용자의 시간을 존중한다 |
| **신뢰성 (Reliable)** | AI 분류 품질이 곧 브랜드 신뢰다. 약속은 반드시 지킨다 |
| **친근함 (Approachable)** | 효율적이되 차갑지 않다. 사용자를 돕는 조력자다 |

---

### 6.2 Do/Don't 예시

| 맥락 | ❌ Don't | ✅ Do | 이유 |
|------|---------|------|------|
| **성공 메시지** | "성공적으로 저장되었습니다!" | "저장됨" | 간결함. 불필요한 감탄사 제거 |
| **오류 메시지** | "AI 분류에 실패했습니다." | "Inbox에 저장됨 — 연결 후 자동 분류" | 긍정적 표현. 해결책 제시 |
| **기능 설명** | "이 기능은 AI가 자동으로 분류합니다." | "AI가 자동으로 정리합니다." | 간결함. "기능은" 불필요 |
| **온보딩** | "Flit.은 혁신적인 메모 앱입니다!" | "떠오른 순간, 바로 던지세요." | 행동 중심. 과장 금지 |
| **일정 제안** | "이 내용을 캘린더에 추가할까요?" | "캘린더에 추가 — 탭하여 승인" | 행동 중심. 질문보다 제안 |
| **재분류 결과** | "AI가 재분류를 완료했습니다." | "재정리 완료 — 3건 이동됨" | 구체적 결과 제시. 수동태 회피 |

---

### 6.3 문장 구조 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **주어 생략 가능** | 맥락이 명확하면 주어 생략 | "오프라인" (O) vs "앱이 오프라인입니다" (X) |
| **능동태 우선** | 수동태는 책임 회피처럼 보임 | "AI가 분류했습니다" (O) vs "분류되었습니다" (X) |
| **상태보다 행동** | 현재 상태가 아닌 다음 행동 안내 | "연결 후 자동 동기화" (O) vs "오프라인 상태" (X) |
| **숫자로 구체화** | 모호한 표현 대신 구체적 수치 | "3건 이동됨" (O) vs "여러 건 이동됨" (X) |

## 8. 브랜드 확장 시나리오

### 8.1 웹사이트 (미래)

- **헤더**: 텍스트 워드마크 "Flit." 좌측 상단
- **히어로 섹션**: "떠오른 순간, 바로 던지면 끝" 태그라인 중앙
- **컬러**: 모노크롬 유지. 배경 #FAFAFA, 텍스트 #111111

---

### 8.2 마케팅 자료

- **앱스토어 스크린샷**: 실제 UI 위주. 로고는 최소화
- **소셜 미디어**: 로고 + 태그라인. 배경은 항상 단색 흰색/검은색
- **프레스킷**: 로고 벡터(SVG), PNG (투명 배경), 컬러/모노크롬 변형 제공

---

### 8.3 브랜드 확장 시 금지사항

- ❌ 로고를 변형하거나 회전하지 않는다
- ❌ 로고 위에 텍스트를 겹치지 않는다
- ❌ 모노크롬 외 컬러(빨강/파랑/녹색 등)를 로고에 추가하지 않는다
- ❌ 로고 주변 여백을 침범하지 않는다 (최소 여백 = 로고 높이의 25%)
- ❌ 워드마크의 채움 도트를 삭제하거나 임의로 크기/위치를 변경하지 않는다
- ❌ 채움 도트를 링(ring) 형태로 변경하지 않는다
