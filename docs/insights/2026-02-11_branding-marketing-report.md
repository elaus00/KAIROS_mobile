## 1. 현재 이름 "KAIROS" 진단

### 1.1 프로덕트-브랜드 적합도: 70/100

KAIROS(카이로스)는 그리스 철학에서 "결정적 순간", "적절한 때"를 뜻한다. "떠오른 순간을 놓치지 않는다"는 철학적 수준에서는 연결되나, 실제 사용자 경험과의 거리가 존재한다.

| 측면 | 평가 |
|------|------|
| "결정적 순간" 의미 | "즉시 기록" 가치와 연결되지만 해석에 한 단계 필요 |
| "Just Capture" 철학과의 정합 | KAIROS는 추상적 개념. "바로 던지면 끝"의 경쾌함과 톤이 다름 |
| 감성적 톤 | 묵직하고 철학적. 앱이 추구하는 "가볍게 던지는" 경험과 무게감 불일치 |
| 브랜드 스토리텔링 | 설명하면 납득되지만, **설명이 필요하다는 것 자체가 약점** |

### 1.2 사용자 인지 (20-30대 한국인)

- **직장인**: "카이로스? 게임 이름 같은데. 뭐하는 앱이지?"
- **대학원생**: "크로노스랑 대비되는 그 개념?" — 소수만 연결 가능
- **ADHD 프리랜서**: 이름 의미보다 기능이 중요. 기억하기 쉬운지가 핵심
- 일반적 연상: 그리스 신화, 교회 용어, 게임/판타지 캐릭터, 고급 브랜드 → "생산성 앱"과의 연결고리 약함

### 1.3 발음/검색성

| 항목 | 평가 |
|------|------|
| 한국어 발음 | "카이로스" 4음절, 발음 자체는 어렵지 않으나 길음 |
| 영문 스펠링 | KAIROS vs KYROS vs CAIROS 혼동 가능 |
| Play Store 검색 | 기존 카이로스 앱, 교회 앱, 게임 등과 경쟁. 독점 검색어 확보 어려움 |
| 글로벌 확장성 | 영어권에서도 niche한 단어 |

### 1.4 경쟁 포지셔닝

| 앱 | 이름 전략 | KAIROS와 비교 |
|------|------|------|
| Todoist | "Todo + ist" — 기능 직접 설명 | KAIROS는 기능을 설명하지 않음 |
| Notion | "생각, 개념" — 추상적이나 핵심 가치와 밀접 | 유사 전략이나 Notion이 더 직관적 |
| Google Keep | "간직하다" — 단순하고 기능적 | KAIROS 대비 기능 연상이 명확 |

**결론**: KAIROS는 "의미를 부여한 이름" 전략으로 Notion/Bear에 가까운 포지션. 이 전략은 브랜드 인지도가 일정 수준 이상일 때 효과적이며, **신규 앱의 초기 발견성/이해도 측면에서는 불리**.

## 2. 네이밍 후보 (22개 분석 → 상위 후보 정리)

### 2.1 생성 방향

철학 가이드에서 도출한 네이밍 기준:
- **"그냥 적으면 된다"** — 무부담, 즉시
- **"기록을 포기하지 않게 만드는 시스템"** — 경쟁자는 "잊어버림"과 "포기"
- **"기록과 정리 사이의 연결고리를 끊는다"** — 분리, 해방
- **공식 용어 = "캡처"** — 메모/입력이 아닌 "캡처"
- **톤**: 간결, 행동 중심, 긍정적, 가벼운

### 2.3 ADHD 비치헤드 관점 재평가

ADHD 사용자의 앱 선택 특성:
- **작업 기억 제한** → 1초 안에 "나한테 필요한 거" 인지 필요
- **생산성 도구 트라우마** → "또 정리하라는 앱이구나" 느낌이면 즉시 이탈
- **"나를 이해하는" 것에 강하게 반응** → 커뮤니티 바이럴 트리거
- **수치심 회피** → "정리 잘하는 앱"보다 "정리 못해도 되는 앱"에 끌림

ADHD 공명 메커니즘 두 가지:

| 전략 | 작동 원리 | 대표 이름 |
|------|-----------|-----------|
| **"너의 고통을 안다"** | 문제를 이름에 담아 즉각적 공감 유발 | **FLIT** |
| **"너의 해결책이다"** | 이미 아는 대처 전략을 이름에 담아 신뢰 유발 | **OFFLOAD** |

## 3. 포지셔닝 전략

### 3.1 핵심 포지션

> **"정리를 포기한 사람을 위한 유일한 앱"**

- 노션 = "정리를 잘하는 사람"을 위한 앱
- 이 앱 = **"정리를 못하는/싫어하는 사람"**을 위한 앱
- 한국 시장에서 이 포지션은 비어 있음

### 3.2 Play Store 메시징

**부제목**: "던지면 AI가 정리해주는 메모"
- 검색 키워드(AI, 메모, 정리) 포함 + 차별점 명확

**스크린샷 5장 구성**:

| 순서 | 헤드라인 | 화면 |
|------|---------|------|
| 1 | "떠오른 순간, 바로 던져" | 캡처 화면 (입력창 즉시 접근) |
| 2 | "AI가 알아서 분류해줘요" | 같은 메모 → 일정/할일/노트 분류 |
| 3 | "카톡 약속도 놓치지 마" | 캘린더 자동 등록 |
| 4 | "쌓인 메모? 정리 부채 끝" | 폴더 자동 정리 전후 비교 |
| 5 | "무료로 시작하세요" | 무료/프리미엄 비교 + CTA |

### 3.3 페르소나별 메시지 차별화

| 타겟 | 메시지 |
|------|--------|
| 일반 사용자 | "던지면 AI가 정리해주는 메모" |
| ADHD 특화 | "정리 못해도 괜찮아. 생각이 사라지기 전에 던져" |
| 직장인 | "카톡 약속, 더블부킹 끝" |
| 대학원생 | "정리 부채 없는 메모 생활" |

## 4. GTM (Go-To-Market) 전략

### 4.1 시장 진입 3단계

| Phase | 기간 | 목표 | 핵심 액션 |
|-------|------|------|-----------|
| 0. 클로즈드 베타 | 출시 전 2주 | 버그 제거 + 리텐션 시그널 | 지인 10~20명 APK 배포, 카톡 오픈채팅 피드백 |
| 1. 소프트 런칭 | 1~4주 | 유저 50명, 리뷰 10개 | 블라인드/에브리타임/ADHD 카페 3개 커뮤니티 공략 |
| 2. 오가닉 확장 | 5~12주 | 지속 성장 엔진 가동 | ASO 반복, 콘텐츠 마케팅, 인플루언서 1~2명 |

### 4.3 ADHD 비치헤드 전략

ADHD 세그먼트가 비치헤드로서 최적인 이유:

| 기준 | ADHD (준혁) | 일반 직장인 (민수) |
|------|------------|------------------|
| 고통의 강도 | "30초 안에 잃어버림" — 생존 수준 | "좀 불편함" — nice-to-have |
| 대안 만족도 | 기존 앱 전부 실패 경험 | "카톡 나에게 보내기로 버팀" |
| 전환 의지 | 즉시 | 낮음 |
| 입소문 속도 | 커뮤니티 결속력 강, 공유 활발 | 개인 사용, 공유 동기 약함 |
| 구독 전환 가능성 | 높음 (절실한 문제 해결) | 낮음 (무료로 충분) |

**실행 액션**:
1. 네이버 ADHD 카페 가입 → 유용한 팁 공유 3회 이상 후 앱 소개
2. Play Store 설명문에 "ADHD 친화적 설계", "정리 부담 제로" 키워드 포함
3. ADHD 특화 콘텐츠: "ADHD인이 메모앱을 포기하는 3가지 이유 (그리고 해결법)"
4. 트위터/X #ADHD #성인ADHD 커뮤니티 참여

### 4.4 바이럴 루프

```
캡처 → AI 분류 → "오 이걸 맞추네!" 놀람 → 공유 카드 생성 → SNS 공유
                                                ↓
                                    주간 리포트 카드 (인스타 스토리 사이즈)
                                                ↓
                                        "나도 써볼래" → 다운로드
```

핵심 트리거:
- AI 분류 정확 시 "공유하기" 버튼 노출
- 주간 성과 카드 자동 생성 (인스타 스토리 사이즈)
- 친구 초대 시 프리미엄 7일 체험 (양쪽 모두)

| 인스타그램 | 감성 + 미니멀 | "정리 못하는 나를 위한 앱" |
| 트위터/X | 위트 + 공감 | "메모앱 3개 깔고 하나도 안 쓰는 사람?" |
| 블라인드 | 실용 + 솔직 | "직장인 메모 문제 해결한 사이드 프로젝트" |
| 네이버 블로그 | 정보성 + SEO | "AI 메모앱 비교 리뷰" |

### 5.4 인플루언서 전략 (예산 0원)

- 팔로워 1,000~10,000 마이크로 인플루언서 타겟
- 카테고리: 생산성 유튜버, 디지털 미니멀리즘 블로거
- 제안: "무료 앱 리뷰 → 프리미엄 평생 무료 제공"
- 초기 목표: 유튜버 1명, 블로거 2명

## 7. Must-Have 기능 & 고도화 영역

### 7.1 Must-Have 기능 (이것 없으면 앱 성립 안 됨)

#### ① 제로 마찰 캡처 — "잡는다"

> ADHD 사용자에게 네비게이션 1단계 = 포기. 캡처까지 3초 넘으면 약속 위반.

| 기능 | 중요도 | 현재 상태 |
|------|--------|-----------|
| 앱 열면 즉시 입력창 | 필수 | 구현됨 |
| **홈 위젯 (원탭 캡처)** | **최우선** | 구현됨 |
| 공유 인텐트 (카톡 → 바로 캡처) | 필수 | 구현됨 |
| 오프라인 저장 (네트워크 무관) | 필수 | 구현됨 |

#### ② AI 분류 정확도 — "정리한다"

> 이 앱의 존재 이유. "던지면 AI가 정리"가 핵심 약속인데, 틀리면 약속이 깨짐.

| 기능 | 중요도 | 현재 상태 |
|------|--------|-----------|
| 유형 자동 판별 (일정/할일/노트/아이디어) | 필수 | 구현됨 |
| AI 제목 생성 | 필수 | 구현됨 |
| 자동 태깅 | 필수 | 구현됨 |
| 낮은 수정 마찰 (1-2탭) | 필수 | 구현됨 |

#### ③ "와, 이걸 맞추네" 모먼트 — "믿게 한다"

> 바이럴 트리거. AI가 정확히 분류한 순간 = 스크린샷을 찍는 순간.

| 기능 | 중요도 | 현재 상태 |
|------|--------|-----------|
| 분류 결과 피드백 (알림 → 분류 현황) | 필수 | 구현됨 |
| 분류 칩 시각적 표시 | 필수 | 구현됨 |

#### ④ 캘린더/투두 자동 연결 — "실행한다"

> "Straight to Action" 원칙의 실체. 기록 → 행동 연결이 가치 실현.

| 기능 | 중요도 | 현재 상태 |
|------|--------|-----------|
| 일정 → Google Calendar 동기화 | 필수 | 구현됨 |
| 할 일 자동 생성 + 완료 처리 | 필수 | 구현됨 |

### 7.2 가장 고도화되어야 하는 영역

#### 1순위: AI 분류 품질 — 앱의 생사를 결정

```
정확도 80% → 불만 ("결국 내가 정리")
정확도 90% → 만족 ("꽤 괜찮네")
정확도 95% → 바이럴 ("이거 마법이다")
```

| 영역 | 현재 | 고도화 방향 |
|------|------|------------|
| 분류 정확도 | 범용 AI 분류 | 수정 이력 학습으로 개인화 (Phase 2b~3b) |
| 한국어 특수성 | 기본 처리 | 카톡 대화체, 줄임말, 이모티콘 정확도 |
| 모호한 입력 | 단일 분류 | 멀티 인텐트 분할 (Phase 2b) |
| 신뢰도 판단 | High/Medium/Low | 임계값 튜닝으로 오등록 최소화 |
| 피드백 루프 | 수정률 추적 | 수정 패턴 → 모델 반영 자동화 |

#### 2순위: 캡처 → 피드백 루프의 속도감

| 구간 | 현재 목표 | 고도화 목표 |
|------|----------|------------|
| 위젯 탭 → 입력 가능 | — | 0.5초 미만 |
| 입력 → 저장 피드백 | 1초 미만 | 유지 |
| 저장 → AI 분류 완료 | 5초 미만 | **3초 미만** |
| 분류 완료 → 사용자 인지 | 알림 | 즉시 푸시 |

#### 3순위: "정리된 느낌"의 시각화

사용자가 **아무것도 안 했는데 정리된 느낌**을 받아야 함. 기능보다 UX 디자인의 문제.

| 영역 | 고도화 포인트 |
|------|-------------|
| 분류 현황 화면 | "오늘 12건 캡처 → 일정 3건, 할일 5건, 노트 4건" 시각적 요약 |
| 캘린더 탭 | 텅 빈 캘린더가 자동으로 채워지는 경험 |
| 노트 폴더 | AI 자동 그룹화한 깔끔한 구조 |
| 주간 리포트 | "이번 주 캡처 42건, 정리 시간 0분" — 정리 부채 제로의 증거 |

### 7.3 한 장 요약

```
[Must-Have 4개]
① 제로 마찰 캡처 (위젯!)    → "잡는다"
② AI 분류 정확도             → "정리한다"
③ "와" 모먼트 (분류 피드백)  → "믿게 한다"
④ 캘린더/투두 자동 연결      → "실행한다"

[고도화 3개]
① AI 분류 품질              → 앱의 생사
② 캡처→피드백 속도           → 습관 형성
③ "정리된 느낌" 시각화       → 리텐션/바이럴
```
